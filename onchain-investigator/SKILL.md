# 链上地址调查分析

## 1. 工作流总览

输入地址 → 判断调查模式 → 识别链 → 并发获取数据 → 交叉分析 → 生成报告

### 调查模式判断

根据用户的描述和地址特征，先判断属于哪种调查模式，因为数据获取策略和分析重点不同：

| 模式 | 触发信号 | 分析重点 |
|------|----------|----------|
| **受害者调查** | "被骗了"、"USDT 没了"、"钱被转走了" | 授权记录、资金去向、受害时间线、是否可追回 |
| **攻击者分析** | "分析这个攻击"、"漏洞利用"、"exploit" | DeFi 持仓结构、交易循环模式、协议交互、损失量化 |
| **地址画像** | "查一下这个地址"、"看看活动" | 资金来源分布、活动时间模式、对手方分析 |

模式不同，第一轮数据请求的优先级也不同：

- **受害者调查** → 优先查授权记录 + 代币转账（找到资金去向）
- **攻击者分析** → 优先查 DeFi 持仓（DeBank）+ 全量交易 + 历史价格
- **地址画像** → 均衡获取，先做全局概览

### 执行策略

1. **快速画像** — 先用 DeBank 30 秒判断调查方向（见 3.0）
2. **判断模式** — 根据画像结果 + 用户描述确定调查类型
3. **识别链** — 根据地址格式判断所属区块链
4. **并发取数据** — oklink（原始交易）+ DeBank（DeFi 持仓、交易语义）并发
5. **补充数据** — TRON 授权追踪 / 历史价格 / 内部交易等
6. **交叉分析** — 关联分析资金流向、对手方地址、异常模式
7. **深挖可疑地址** — 对可疑对手方地址递归查询（控制在 2-3 跳，避免发散）
8. **量化损失** — 结合历史价格计算精确的 USD 损失
9. **生成报告** — 按标准模板输出结构化调查报告
