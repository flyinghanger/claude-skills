# Think Deeper - 提示词优化

## 核心理念

用户脑中有想法，但表达出来往往是模糊的、不完整的。**主动思考「如果我要执行这个任务，还需要知道什么？」**，然后把问题抛给用户，帮他们把「我大概想做个XX」变成清晰可执行的任务描述。

## 工作流程

### Phase 1：解析原始意图

收到用户描述后：

1. **一句话复述**你理解的核心意图，让用户确认方向
2. **项目上下文扫描**（强制）：如果在项目目录中，**必须先** Glob/Grep 扫描项目结构、技术栈、已有实现，用扫描结果填充已知信息。禁止问项目中已经存在的信息（如技术栈、框架版本、目录结构等）
3. **5 维快速评分**：用表格展示用户描述在 5 个维度的评分（参考 `references/clarity-dimensions.md` 中的评分标准）

| 维度 | 评分 | 说明 |
|------|------|------|
| 清晰度 | ?/10 | |
| 具体性 | ?/10 | |
| 上下文 | ?/10 | |
| 完整性 | ?/10 | |
| 结构性 | ?/10 | |

评分让用户直观看到哪里弱，也为后续提问提供方向依据。

4. **快速识别**已知信息和模糊/缺失信息（参考 `references/clarity-dimensions.md`）

### Phase 2：主动提问

从缺失维度中挑选最关键的问题提问。
